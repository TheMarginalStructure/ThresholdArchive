# 阈界分类标准

---

## 1.0 概述

本文件规定了边际结构（The Marginal Structure）内部阈界现象的分类标准，用于对阈界和异常现象进行系统化分类，确保档案编码、研究和应对措施的准确性与一致性。分类标准适用于所有与阈界相关的档案，包括勘探报告（EL）、对象档案（TF）、医疗报告（MHR）、理论文件（TD）等。

**目标**：
- 提供统一的阈界分类框架，支持档案管理和研究。
- 确保分类的全面性、系统性和可扩展性。
- 符合边际结构的世界观设定和档案管理要求。

**适用范围**：
- 外勤行动部、档案与研究部、医疗评估部等所有处理阈界相关档案的部门。
- 所有活跃、归档或受限访问状态的阈界记录。

## 3.0 分类规则总则

阈界分类代码由以下部分组成：

1. **大类代码**：1个大写字母，表示阈界的主要类型（如L、O、V）。
2. **子类别代码**（可选）：3个字母，细化阈界特性（如LND、SPC），与《档案编码规则.md》的子类别代码一致。
3. **序列号**：4位数字，用于区分同一阈界的多个实例（如0001）。

**格式**：
- 大类代码：`[大类代码]-[序列号]`（如L-234）
- 带子类别：`[大类代码]-[序列号]-[子类别代码]`（如TF-L234-LND）

**通用规则**：
- 大类代码为单一大写字母，优先使用高频异常类型的字母。
- 子类别代码参考《档案编码规则.md》，仅在需要细化时使用。
- 序列号从001开始，递增至9999，超出时需申请新大类代码。
- 分类代码需在归档前由档案与研究部审核，确保唯一性和合规性。
- 涉及敏感阈界（如V、O）的档案需加密存储，访问权限不得低于3级。

## 4.0 阈界大类代码

以下为边际结构阈界现象的主要大类代码及其说明。

| **代码** | **大类** | **说明** |
|----------|----------|----------|
| **L** | Location-based（基于地点/空间） | 基于特定地点或空间的阈界，如异常山脉、建筑、非欧几里得空间。 |
| **O** | Ontological（存在性/概念性） | 涉及存在性、哲学或抽象概念的阈界，如“不可知知识”或现实扭曲。 |
| **V** | Viral/Memetic（模因/传播性） | 通过信息传播引发异常的阈界，如模因危害、认知传染。 |
| **E** | Entity-specific（特定实体） | 与特定异常实体相关的阈界，如超自然生物、意识实体。 |
| **T** | Temporal（时间异常） | 涉及时间扭曲、时间循环或时间异常的阈界。 |
| **P** | Physical（物理异常） | 违反已知物理规律的阈界，如反重力、物质转化。 |
| **B** | Biological（生物异常） | 涉及异常生物或生物变异的阈界，如变异植物、异常动物。 |
| **A** | Artifact（人工制品/技术异常） | 涉及异常设备或技术的阈界，如自运行机器、异常AI。 |

**注意事项**：
- 大类代码需与档案编码中的阈界代码一致（如TF-L234-LND）。
- 新增大类代码需经档案与研究部申请，总指挥（伊利亚·彼得连科）批准。
- 涉及V（模因）或O（存在性）的阈界需加密存储，访问权限不得低于3级。

## 5.0 子类别代码

子类别代码用于细化阈界特性，与《档案编码规则.md》的子类别代码一致，仅在需要时附加至档案编码。以下为常见子类别代码及其适用范围。

| **代码** | **描述** | **适用大类** | **说明** |
|----------|----------|--------------|----------|
| LND | 地形（Landform） | L | 涵盖山脉、森林、大海等自然地形异常。 |
| SPC | 空间异常（Spatial Anomaly） | L, T | 非欧几里得空间、传送现象等。 |
| TMP | 时间异常（Temporal Anomaly） | T | 时间扭曲、时间循环等。 |
| COG | 认知危害（Cognitive Hazard） | O, V | 影响认知或意识的异常。 |
| MEM | 模因危害（Memetic Hazard） | V | 通过信息传播引发的危害。 |
| ENT | 实体（Entity） | E | 超自然生物、意识实体等。 |
| OBJ | 物体（Object） | A, P | 异常物体，如自运行装置。 |
| BIO | 生物异常（Biological Anomaly） | B | 异常生物或生物变异。 |
| PSY | 心理异常（Psychological Anomaly） | O, V | 非模因的心理影响，如情绪操控。 |
| TEC | 技术异常（Technological Anomaly） | A | 异常设备或技术，如异常AI。 |

**注意事项**：
- 子类别代码在对象档案（TF）中必填，在其他档案类型（如EL、MHR）中可选。
- 新增子类别代码需参考《档案编码规则.md》，由档案与研究部审核。
- 子类别代码需反映阈界的具体特性，避免过于宽泛或具体。

## 6.0 分类分配与管理

**分配流程**：
1. **申请**：档案生成部门提交阈界分类申请，包含大类代码、子类别代码（如适用）和阈界描述。
2. **审核**：档案与研究部核对《阈界分类标准》和《档案编码规则.md》，确保代码唯一性和合规性。
3. **分配**：生成阈界代码（如L-234），记录至《阈界登记数据库》。
4. **存档**：代码与档案内容一同归档，需附档案员签名。

**管理要求**：
- 所有阈界代码需录入《阈界登记数据库》，包含生成日期、档案员和访问权限。
- 代码冲突需立即报告档案与研究部，重新分配。
- 序列号超出9999时，需申请新大类代码或子类别代码。
- 归档后代码不得更改，除非经总指挥（伊利亚·彼得连科）批准。
- 历史阈界代码（不符合本标准）需重新分配，原代码记录至“关联记录”字段。

**加密与访问**：
- 涉及V（模因）、O（存在性）或PSY（心理异常）的阈界需加密存储，访问权限不得低于3级。
- 阈界代码需在档案信息表格中明确标注，遵循《综合档案模板.md》。

## 8.0 注意事项

1. **一致性**：阈界代码需与《档案编码规则.md》和《边际结构组织架构表》保持一致。
2. **保密性**：涉及V、O、PSY的阈界需加密存储，访问权限严格控制。
3. **可扩展性**：大类代码保留未使用字母（如C、D、F），支持未来扩展。
4. **错误处理**：发现代码错误（如重复或格式不符）需立即报告档案员（安雅·夏尔马），由档案与研究部修正。
5. **历史阈界**：早期阈界代码（如不符合本标准）需重新分配，记录原代码至“关联记录”字段。
6. **审核周期**：分类标准需每12个月审查一次，确保适应组织需求。
