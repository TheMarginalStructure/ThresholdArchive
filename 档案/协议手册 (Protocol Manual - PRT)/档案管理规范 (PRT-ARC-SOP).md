# 档案管理规范

---

## 1.0 概述

本文件规定了边际结构（The Marginal Structure）内部档案管理的全面规范，涵盖档案编码、分类、创建、审核、存储、访问、更新和销毁的流程与要求。目标是确保档案的系统性、一致性、可追溯性和安全性，支持组织对阈界和异常现象的研究与应对。规范适用于所有档案类型，包括探索报告（EL）、对象档案（TF）、医疗报告（MHR）、理论文件（TD）、实验记录（EXP）、音频/视频记录（AUD/VID）、设备日志（EQP）和行政文件（ADM）。

**目标**：
- 确保档案编码和分类的统一性，方便检索和跨部门协作
- 保障档案的安全性，防止敏感信息泄露
- 提供清晰的管理流程，支持档案全生命周期管理
- 符合边际结构的世界观设定和组织架构要求

**适用范围**：
- 外勤行动部、研究与档案部、医疗评估部等所有生成或使用档案的部门
- 所有活跃、归档或受限访问状态的档案
- 档案管理相关的所有人员和流程

## 3.0 总则/核心原则

档案管理遵循统一性、安全性、可追溯性和完整性原则，确保组织信息资产的有效管理和保护。

**通用规则**：
- 所有档案必须按照统一编码规则进行标识和分类
- 档案访问权限严格按照分级制度执行
- 档案生命周期管理流程必须完整记录
- 敏感档案必须采用相应的加密和保护措施

**注意事项**：
- 编码分配必须确保唯一性，避免冲突
- 档案更新必须保留版本历史
- 档案销毁需要最高级别授权
- 定期审核档案状态和访问权限

## 4.0 档案编码规则

档案编码是档案管理的核心标识系统，确保每个档案都有唯一的识别码。

### 4.1 编码格式

档案编码由以下部分组成，使用连字符（-）分隔：

1. **档案类型代码**：2-3个字母，表示档案类型（如EL、TF）
2. **部门/队伍代码**（部分类型适用）：如Gamma（勘探队）、ARC（研究与档案部）
3. **阈界/对象代码**（部分类型适用）：如L-234，基于《阈界分类标准》
4. **序列号**：4位数字，从0001至9999
5. **子类别代码**（可选或必填）：3个字母，如LND（地形）、COG（认知危害）

**格式**：`[档案类型]-[部门/队伍代码（可选）]-[阈界/对象代码（可选）]-[序列号]-[子类别代码（可选）]`

### 4.2 档案类型编码

| **档案类型** | **编码格式** | **示例** |
|--------------|--------------|----------|
| 探索报告（EL） | EL-[队伍代码]-[阈界代码]-[序列号]-[子类别代码（可选）] | EL-Gamma-L234-COG |
| 对象档案（TF） | TF-[阈界代码]-[序列号]-[子类别代码] | TF-L234-LND |
| 医疗报告（MHR） | MHR-[阈界代码]-[序列号]-[子类别代码（可选）] | MHR-L234-COG |
| 理论文件（TD） | TD-[阈界代码]-[序列号]-[子类别代码] | TD-L234-MEM |
| 实验记录（EXP） | EXP-[阈界代码]-[序列号]-[子类别代码（可选）] | EXP-A512-BIO |
| 音频/视频记录（AUD/VID） | [AUD/VID]-[阈界代码]-[序列号] | AUD-L234 |
| 设备日志（EQP） | EQP-[设备类型]-[序列号] | EQP-MkIV |
| 行政文件（ADM） | ADM-[部门代码]-[序列号] | ADM-ARC-0003 |

**示例**：
探索报告编码示例：EL-Gamma-L234-0001-COG 表示伽马队对明知山（L-234）的第一份探索报告，涉及认知危害。

## 5.0 阈界分类标准

阈界分类标准为档案编码提供统一的对象识别体系。

### 5.1 大类代码

阈界分类代码由1个大写字母（大类代码）和4位序列号组成，子类别代码可选。

| **代码** | **大类** | **说明** |
|----------|----------|----------|
| L | Location-based | 基于地点/空间的阈界，如山脉、非欧几里得空间 |
| O | Ontological | 存在性/概念性阈界，如抽象概念、现实扭曲 |
| V | Viral/Memetic | 模因/传播性现象，如信息传染 |
| E | Entity-specific | 特定实体，如异常生物、意识实体 |
| T | Temporal | 时间异常，如时间循环、时间扭曲 |
| P | Physical | 违反物理规律的异常，如反重力 |
| B | Biological | 异常生物或生物变异 |
| A | Artifact | 异常设备或技术，如异常AI |

### 5.2 子类别代码

| **代码** | **描述** | **适用大类** |
|----------|----------|-------------|
| LND | 地形（山脉、森林、大海等） | L |
| SPC | 空间异常 | L, T |
| TMP | 时间异常 | T |
| COG | 认知危害 | O, V |
| MEM | 模因危害 | V |
| ENT | 实体 | E |
| OBJ | 物体 | A, P |
| BIO | 生物异常 | B |
| PSY | 心理异常 | O, V |
| TEC | 技术异常 | A |

**流程**：
1. **申请**：档案生成部门提交编码申请，包含档案类型、阈界代码、队伍/部门代码和子类别
2. **审核**：研究与档案部核对《阈界分类标准》和《边际结构组织架构表》，确保唯一性
3. **分配**：生成编码，记录至《档案登记数据库》
4. **存档**：编码与档案一同归档，附档案员签名

**要求**：
- 编码长度不得超过32个字符，禁止使用特殊字符或空格
- 子类别代码在TF和TD中必填，其他类型可选
- 序列号超出9999时，需申请新阈界代码或子类别代码

## 6.0 档案管理流程

档案管理流程涵盖档案的完整生命周期，从创建到销毁的全过程管理。

### 6.1 档案创建

**责任部门**：生成部门（如外勤行动部、医疗评估部）负责档案初稿。

**要求**：
- 使用《综合档案模板.md》中对应类型的模板（如EL、TF）
- 填写档案信息表格，包含归档编码、威胁等级、访问权限等
- 确保人员信息符合《边际结构组织架构表》，使用标准姓名格式（如米拉·陈、戴维·卡特博士）

### 6.2 档案审核

**责任人**：档案员（安雅·夏尔马）或指定审核人。

**流程**：
1. 检查档案格式是否符合《综合档案模板.md》
2. 核对编码是否符合《档案编码规则》
3. 验证阈界代码是否符合《阈界分类标准》
4. 确认敏感信息是否加密

### 6.3 档案存储与访问

**存储要求**：
- 普通档案：存储于《档案登记数据库》，访问权限1-2级
- 敏感档案（COG、MEM、PSY）：加密存储于安全服务器，访问权限3级及以上，加密等级为高
- 物理档案（如样本）：存储于C级实验室，符合《实验室安全操作规程》

**访问权限分级**：

| **级别** | **权限** | **可访问档案类型** |
|----------|----------|---------------------|
| 1级 | 公开 | ADM（部分）、EQP（部分） |
| 2级 | 内部 | ADM、EQP、TD（部分） |
| 3级 | 受限 | EL、TF、MHR、EXP、AUD/VID |
| 4级 | 高度受限 | 涉及COG、MEM、PSY的档案 |
| 5级 | 仅限高层 | 总指挥（伊利亚·彼得连科）批准的档案 |

## 7.0 示例场景/应用案例

以下为典型档案管理场景，展示规范的实际应用。

**场景1：明知山（L-234）档案管理**
- **情况**：伽马队完成明知山勘探任务，需要创建相关档案
- **应用**：按照编码规则生成EL-Gamma-L234-COG、TF-L234-LND、MHR-L234-COG等档案
- **示例**：敏感数据（COG）加密存储，访问权限3级，定期审查（6个月），更新生物计量数据

**场景2：时间异常（T-789）档案管理**
- **情况**：发现时间异常现象，需要建立完整的档案体系
- **应用**：创建EL-Sigma-T789-TMP、TD-T789-TMP等档案，实验记录加密存储
- **示例**：访问权限4级，发现时间循环数据时更新档案，生成新版本

**场景3：档案编码冲突处理**
- **情况**：发现同一编码被多个档案使用
- **应用**：暂停相关档案使用，档案员重新分配编码，更新数据库
- **示例**：原编码记录至"关联文件"字段，确保可追溯性

## 8.0 注意事项

1. **编码唯一性**：确保每个档案编码的唯一性，避免重复分配
2. **权限管理**：严格按照访问权限分级制度执行，定期审核权限设置
3. **版本控制**：档案更新时必须保留原版本，记录更新日志
4. **加密要求**：敏感档案必须采用相应级别的加密措施
5. **审核周期**：按照规定周期进行档案审核，及时更新状态
6. **备份管理**：所有档案需在归档后24小时内备份，确保数据安全
