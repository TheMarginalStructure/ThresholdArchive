# 数据库搭建与数据导入指南

> 用于在新电脑上搭建 PostgreSQL 数据库并导入 ThresholdArchive 项目数据。
> 将此文件 + `backend/seed-data-export.json` 交给 Reasonix 即可一键完成配置。

---

## 前置条件

- Node.js >= 18
- Git
- SSH 已按 `设定/SSH配置迁移指南.md` 配置完毕

---

## 第一步：克隆仓库

```bash
cd D:/Github
git clone git@tms:TheMarginalStructure/ThresholdArchive.git
cd ThresholdArchive
```

---

## 第二步：安装 PostgreSQL

### 下载
从 https://www.postgresql.org/download/windows/ 下载 PostgreSQL 安装包（推荐 16.x 版本）。

### 安装
一路默认下一步，注意：
- **密码**：设置为 `915049`（与项目配置一致）
- **端口**：默认 `5432`
- 安装完成后，勾选 **Launch Stack Builder** 的选项可以取消

### 验证安装
```bash
psql -U postgres -d postgres -c "SELECT version();"
# 输入密码 915049，应显示 PostgreSQL 版本信息
```

---

## 第三步：创建数据库

```powershell
# 连接 PostgreSQL
psql -U postgres -p 5432

# 输入密码: 915049

# 创建数据库
CREATE DATABASE threshold_archive;

# 查看确认
\l

# 退出
\q
```

---

## 第四步：安装依赖并生成 Prisma Client

```bash
cd D:/Github/ThresholdArchive

# 后端依赖
npm --prefix backend install

# 生成 Prisma Client
cd backend
npx prisma generate
cd ..
```

> 注意：如果 `npx prisma generate` 报错找不到 schema，指定路径：
> ```bash
> npx prisma generate --schema=backend/prisma/schema.prisma
> ```

---

## 第五步：导入数据库 Schema + 数据

### 方式 A：一键导入（推荐）

```bash
# 先创建表结构
npx prisma db push --schema=backend/prisma/schema.prisma

# 导入数据
node backend/scripts/import-db.js
```

### 方式 B：从种子数据重建

```bash
# 如果只想要 seed 数据的初始状态（不含 CMS 修改），运行：
npx prisma db push --schema=backend/prisma/schema.prisma
npx tsx backend/prisma/seed.ts
```

---

## 第六步：配置环境变量

创建 `backend/.env` 文件：

```env
DATABASE_URL="postgresql://postgres:915049@localhost:5432/threshold_archive?schema=public"
PORT=3001
JWT_SECRET="threshold-archive-secret-key-2026"
NODE_ENV=development
```

---

## 第七步：验证

```bash
# 启动后端
npm --prefix backend run dev

# 另开终端，测试 API 是否正常返回数据
curl http://localhost:3001/api/v1/archives
```

如果返回 JSON 格式的档案列表，说明数据库搭建成功。
