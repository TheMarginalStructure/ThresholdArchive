# 档案管理规范

---

## 1.0 概述

本文件规定了边际结构（Threshold Structure）内部档案管理的全面规范，涵盖档案编号、分类、创建、审核、存储、访问、更新和销毁的流程与要求。目标是确保档案的系统性、一致性、可追溯性和安全性，支持组织对阈界和异常现象的研究与应对。规范适用于所有档案类型，包括探索报告（EL）、对象档案（TF）、医疗报告（MHR）、理论文件（TD）、实验记录（EXP）、音频/视频记录（AUD/VID）、设备日志（EQP）和行政文件（ADM）。

**目标**：
- 确保档案编号和分类的统一性，方便检索和跨部门协作。
- 保障档案的安全性，防止敏感信息泄露。
- 提供清晰的管理流程，支持档案全生命周期管理。
- 符合边际结构的世界观设定和组织架构要求。

**适用范围**：
- 外勤行动部、研究与档案部、医疗评估部等所有生成或使用档案的部门。
- 所有活跃、归档或受限访问状态的档案。

## 3.0 档案编号规则

### 3.1 编号格式

档案编号由以下部分组成，使用连字符（-）分隔：

1. **档案类型代码**：2-3个字母，表示档案类型（如EL、TF）。
2. **部门/队伍代码**（部分类型适用）：如Gamma（勘探队）、RES（研究与档案部）。
3. **阈界/对象代码**（部分类型适用）：如L-234，基于《阈界分类标准》。
4. **序列号**：4位数字，从0001至9999。
5. **子类别代码**（可选或必填）：3个字母，如LND（地形）、COG（认知危害）。

**格式**：`[档案类型]-[部门/队伍代码（可选）]-[阈界/对象代码（可选）]-[序列号]-[子类别代码（可选）]`

### 3.2 档案类型编号

| **档案类型** | **编号格式** | **示例** |
|--------------|--------------|----------|
| 探索报告（EL） | EL-[队伍代码]-[阈界代码]-[序列号]-[子类别代码（可选）] | EL-Gamma-L234-0001-COG |
| 对象档案（TF） | TF-[阈界代码]-[序列号]-[子类别代码] | TF-L234-0001-LND |
| 医疗报告（MHR） | MHR-[阈界代码]-[序列号]-[子类别代码（可选）] | MHR-L234-0001-COG |
| 理论文件（TD） | TD-[阈界代码]-[序列号]-[子类别代码] | TD-L234-0001-MEM |
| 实验记录（EXP） | EXP-[阈界代码]-[序列号]-[子类别代码（可选）] | EXP-A512-0001-BIO |
| 音频/视频记录（AUD/VID） | [AUD/VID]-[阈界代码]-[序列号] | AUD-L234-0001 |
| 设备日志（EQP） | EQP-[设备类型]-[序列号] | EQP-MkIV-0001 |
| 行政文件（ADM） | ADM-[部门代码]-[序列号] | ADM-RES-0003 |

**注意事项**：
- 编号长度不得超过32个字符，禁止使用特殊字符或空格。
- 子类别代码在TF和TD中必填，其他类型可选。
- 序列号超出9999时，需申请新阈界代码或子类别代码。

### 3.3 编号分配流程

1. **申请**：档案生成部门提交编号申请，包含档案类型、阈界代码、队伍/部门代码和子类别。
2. **审核**：研究与档案部核对《阈界分类标准》和《边际结构组织架构表》，确保唯一性。
3. **分配**：生成编号，记录至《档案登记数据库》。
4. **存档**：编号与档案一同归档，附档案员签名。

## 4.0 阈界分类标准

### 4.1 大类代码

阈界分类代码由1个大写字母（大类代码）和4位序列号组成，子类别代码可选。

| **代码** | **大类** | **说明** |
|----------|----------|----------|
| L | Location-based | 基于地点/空间的阈界，如山脉、非欧几里得空间。 |
| O | Ontological | 存在性/概念性阈界，如抽象概念、现实扭曲。 |
| V | Viral/Memetic | 模因/传播性现象，如信息传染。 |
| E | Entity-specific | 特定实体，如异常生物、意识实体。 |
| T | Temporal | 时间异常，如时间循环、时间扭曲。 |
| P | Physical | 违反物理规律的异常，如反重力。 |
| B | Biological | 异常生物或生物变异。 |
| A | Artifact | 异常设备或技术，如异常AI。 |

### 4.2 子类别代码

子类别代码细化阈界特性，与档案编号一致。

| **代码** | **描述** | **适用大类** |
|----------|----------|--------------|
| LND | 地形（山脉、森林、大海等） | L |
| SPC | 空间异常 | L, T |
| TMP | 时间异常 | T |
| COG | 认知危害 | O, V |
| MEM | 模因危害 | V |
| ENT | 实体 | E |
| OBJ | 物体 | A, P |
| BIO | 生物异常 | B |
| PSY | 心理异常 | O, V |
| TEC | 技术异常 | A |

**注意事项**：
- 子类别代码在对象档案（TF）中必填，其他档案类型可选。
- 新增大类或子类别需经研究与档案部审核，总指挥（伊利亚·彼得连科）批准。

## 5.0 档案管理流程

### 5.1 档案创建

- **责任部门**：生成部门（如外勤行动部、医疗评估部）负责档案初稿。
- **要求**：
  - 使用《综合档案模板.md》中对应类型的模板（如EL、TF）。
  - 填写档案信息表格，包含归档编号、威胁等级、访问权限等。
  - 确保人员信息符合《边际结构组织架构表》，使用标准姓名格式（如米拉·陈、戴维·卡特博士）。
- **示例**：伽马队生成探索报告（EL-Gamma-L234-0001-COG），需包含任务背景、数据分析和签名确认。

### 5.2 档案审核

- **责任人**：档案员（安雅·夏尔马）或指定审核人。
- **流程**：
  1. 检查档案格式是否符合《综合档案模板.md》。
  2. 核对编号是否符合《档案编号规则》。
  3. 验证阈界代码是否符合《阈界分类标准》。
  4. 确认敏感信息是否加密。
- **审核周期**：归档后7日内完成审核，重大档案需总指挥（伊利亚·彼得连科）批准。

### 5.3 档案存储

- **存储要求**：
  - 普通档案：存储于《档案登记数据库》，访问权限1-2级。
  - 敏感档案（COG、MEM、PSY）：加密存储于安全服务器，访问权限3级及以上，加密等级为高。
  - 物理档案（如样本）：存储于C级实验室，符合《实验室安全操作规程》。
- **备份**：所有档案需在归档后24小时内备份，备份编号记录至档案信息表格。

### 5.4 档案访问

- **访问权限分级**：
  | **级别** | **权限** | **可访问档案类型** |
  |----------|----------|---------------------|
  | 1级 | 公开 | ADM（部分）、EQP（部分） |
  | 2级 | 内部 | ADM、EQP、TD（部分） |
  | 3级 | 受限 | EL、TF、MHR、EXP、AUD/VID |
  | 4级 | 高度受限 | 涉及COG、MEM、PSY的档案 |
  | 5级 | 仅限高层 | 总指挥（伊利亚·彼得连科）批准的档案 |
- **访问流程**：
  1. 提交访问申请，包含档案编号和访问理由。
  2. 研究与档案部审核，确认申请人权限。
  3. 敏感档案需额外验证身份，记录访问日志。

### 5.5 档案更新

- **更新触发**：
  - 新数据或事件（如二次勘探）。
  - 编号或分类规则变更。
  - 定期审查发现问题。
- **流程**：
  1. 提交更新申请，说明变更内容。
  2. 档案员审核，更新《档案登记数据库》。
  3. 保留原档案版本，记录更新日志。
- **示例**：TF-L234-0001-LND更新为包含新观测数据，生成新版本编号（如TF-L234-0001-LND-v2）。

### 5.6 档案销毁

- **销毁条件**：
  - 档案威胁等级降为无，且无研究价值。
  - 存在模因危害，需销毁以防止传播。
- **流程**：
  1. 提交销毁申请，需总指挥（伊利亚·彼得连科）批准。
  2. 销毁前备份元数据至《档案登记数据库》。
  3. 物理销毁使用C级实验室设施，电子销毁使用安全擦除协议。
- **记录**：销毁记录存档，编号为ADM-RES-[序列号]-DST。

## 6.0 错误处理与异常情况

### 6.1 编号冲突

- **情况**：同一编号被多个档案使用。
- **处理**：
  1. 暂停相关档案使用，通知生成部门。
  2. 档案员重新分配编号，更新《档案登记数据库》。
  3. 原编号记录至“关联文件”字段。

### 6.2 分类错误

- **情况**：阈界代码或子类别代码不符合《阈界分类标准》。
- **处理**：
  1. 档案员核查《阈界分类标准》，重新分配代码。
  2. 更新相关档案，记录原代码至“关联记录”字段。

### 6.3 档案丢失

- **情况**：档案无法在数据库或存储中找到。
- **处理**：
  1. 报告总指挥（伊利亚·彼得连科），启动调查。
  2. 尝试从备份恢复，若无法恢复，生成丢失报告（ADM-RES-[序列号]-LSS）。
  3. 更新《档案登记数据库》，标记为“丢失”。
